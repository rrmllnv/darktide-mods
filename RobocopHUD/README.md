# RoboCopHUD

HUD‑мод в стиле интерфейса RoboCop: “сканирование”, “захват цели”, системные директивы и телеметрия, которые ощущаются как работа полицейского киборга, а не как обычный игровой HUD.

## Цели

- Сделать **визор/интерфейс** главным “персонажем”: HUD не просто показывает цифры, а **ищет**, **оценивает**, **подтверждает** и **сопровождает**.
- Дать узнаваемые элементы: **стрелка наведения на цель**, **SCAN → TRACK → LOCK**, **DIRECTIVES**, **SYSTEM WARNINGS**, **REC + timestamp**, “OCR/ID” карточка.
- Отдельный стиль: единая палитра, типографика, сетки/рамки, анимации “механики”.

## Концепция экранных зон (layout)

- **Center (reticle area)**: захват/рамка цели, статус “SCAN/TRACK/LOCK”, короткие подтверждения (“LOCK CONFIRMED”).
- **Right column**: матрица угроз (TOP THREATS), краткие карточки 3–6 целей.
- **Left column**: телеметрия и диагностика (POWER/TEMP/SERVO/GYRO/NET) + состояние брони/перегруза/подавления.
- **Top bar**: REC + таймкод, режим визора, индикатор “SCAN SWEEP”.
- **Bottom bar**: директивы/протоколы (DIRECTIVE ACTIVE), контекстные подсказки, after‑action протокол.

## Основные виджеты (MVP)

- **Target Acquisition Arrow**
  - Стрелка/указатель, который “ищет” цель и ведёт взгляд.
  - Режимы: nearest hostile / tagged hostile / objective / ally in distress.
  - Анимации: “search sweep”, “snap to target”, “lost + reacquire”.

- **Lock‑On Frame**
  - Рамка вокруг цели с фазами: `SCAN → TRACK → LOCK`.
  - Параметры: время захвата, удержание после потери LOS, плавность.
  - Визуальный “механический” фидбек: сегменты заполняются, “степы” фиксации.

- **Threat Ladder (TOP THREATS)**
  - Список приоритетных угроз (3–6) с индексом риска и дистанцией/направлением.
  - Приоритет: special > elite > boss > ближайший > агрессивный.
  - Стабилизация списка (анти‑фликер): таймер удержания позиции/цели.

- **OCR / ID Card**
  - Мини‑карточка текущей цели: `ID`, `CLASS`, `RISK`, `STATUS`, `LAST SEEN`.
  - Фейковая “классификация” (визуальный стиль) + реальные признаки (тип, состояние, distance, LOS).

- **Sweep Radar**
  - Дуга/линия сканирования, которая раз в N секунд “пингует” окружение.
  - Варианты отображения: точки по секторам, дальность, плотность.
  - Режимы: пассивный / активный (по нажатию).

- **Recorder Overlay**
  - `REC ●` + таймкод, отметки событий (“EVIDENCE CAPTURED”, “DIRECTIVE UPDATED”).
  - Опциональная “вставка” кадровой сетки/уровня горизонта.

## Системные предупреждения (вместо обычных алертов)

Сделать словарь “киборг‑алертов”, которые генерятся из игровых событий и статусов:

- **ARMOR BREACH**: резкое падение защиты/здоровья, пробитие.
- **SENSOR JAM**: сильные визуальные эффекты/подавление/ослепление (если можно детектить).
- **AUDIO OVERLOAD**: пик событий/взрывов/подавления (как стилизация).
- **TARGET LOST**: потеря текущей цели.
- **HOSTAGE RISK**: союзник схвачен/в сетке/под собакой (если доступно).
- **LOW AMMO / LOW POWER**: ресурсы на минимуме (аммо/гранаты/способность).

## Директивы и “протокол”

- **DIRECTIVES (1–3 строки)**: меняются по контексту (бой/объектив/опасность/команда).
- **RULE‑BASED UI**: интерфейс принимает “решения” и показывает причины (“PRIORITY: SPECIALIST (LOS+RANGE)”).
- **After‑Action Brief**: краткий отчёт после стычки: threats neutralized / damage taken / time.

## Визор‑стили (темы)

- **Classic Green**: зелёная палитра, “старый CRT” стиль.
- **Amber Tactical**: янтарный/оранжевый, много предупреждений.
- **Police Blue/White**: холодный бело‑синий “служебный” интерфейс.

Для каждой темы:
- базовые цвета (primary/secondary/alert/success)
- шрифты/толщина линий/прозрачности
- вариант сетки/рамки

## Анимации и “ощущение механики”

- Сегментные прогресс‑индикаторы (ступенчатые, не гладкие).
- Микро‑задержки подтверждения (визуальный “компьютер думает”).
- Пульс/скан‑проход по рамке цели.
- “Freeze hold”: удержание меток на 0.5–1.5с после потери.

## Интерактивность (опционально)

- **Manual Scan**: кнопка запуска активного скана.
- **Cycle Targets**: переключение цели по списку угроз.
- **Pin Target**: “закрепить” цель на время.
- **Mode Switch**: Combat / Scanner / Protocol.

## Настройки мода (план меню)

- **General**
  - включение HUD, масштаб, opacity, позиционирование
  - тема/палитра
  - debug overlay

- **Targeting**
  - режим выбора цели
  - скорость стрелки, агрессивность “липкости”, удержание после потери
  - пороги для LOCK (время/дистанция/LOS)

- **Threat Matrix**
  - количество слотов, анти‑фликер, правила приоритета
  - скрывать/показывать классы, дистанцию, статус

- **Scanner**
  - частота sweep, дальность, тип визуализации
  - активный/пассивный режим

- **Warnings**
  - какие предупреждения включены
  - cooldown/спам‑защита

## Технический план реализации (черновик)

- **Архитектура**
  - отдельный HUD‑элемент + независимые виджеты (arrow/lock/threat/scanner/warnings)
  - отдельный runtime слой для “выбора цели” и “оценки угроз”
  - отдельный слой стилей/тем

- **Поток данных**
  - сбор кандидатов целей → оценка (score) → выбор текущей цели → отрисовка
  - события: появление/исчезновение, потеря LOS, смена режима, ручной скан
  - анти‑фликер: hysteresis + time‑based smoothing

- **Производительность**
  - лимиты на количество кандидатов/частоту пересчёта
  - кэширование результатов и “ленивые” обновления UI
  - строгий контроль аллокаций в update/draw

## Роадмап (по этапам)

- **Phase 1 (MVP)**: Arrow + Lock‑On + базовая OCR карточка + тема + REC overlay.
- **Phase 2**: Threat Ladder + sweep‑сканер + warnings.
- **Phase 3**: Режимы (Combat/Scanner/Protocol) + ручные действия + after‑action.
- **Phase 4**: Полировка: анимации “механики”, тонкая типографика, расширение правил угроз.
